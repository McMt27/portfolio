SAE 1.04 - Site Web BMW

Résumé du projet

Dans le cadre de la SAE 1.04, j’ai réalisé un site web statique autour d’un thème personnel : BMW. Ce projet avait pour but de valider les compétences en HTML, CSS, accessibilité, design responsive et structuration de code. Le site est organisé autour de plusieurs pages (accueil, modèles, à propos, etc.) avec une charte graphique cohérente, des animations CSS et un hébergement sur GitHub Pages.

Apprentissages critiques

Voici les compétences développées durant cette SAE :

Compétence Description Statut
AC13.01 Utiliser un système informatique et ses outils Validée
AC13.04 Connaître l’architecture et les technologies d’un site Web Validée

Ce que j'ai fait précisément dans la SAE 1.04

  • Création complète d’un site HTML/CSS
  • Utilisation de Flexbox et CSS Grid pour la mise en page
  • Ajout d’animations CSS sur les titres et les boutons
  • Application d’une charte graphique cohérente et responsive
  • Respect des standards W3C HTML/CSS et de la norme d’accessibilité WCAG 2.0 AA
  • Validation du code via les validateurs W3C
  • Dépôt du projet sur GitHub avec commits réguliers et déploiement GitHub Pages

Retour d'expérience

Points forts du projet

  • Site bien structuré et clair visuellement
  • Animations CSS efficaces et bien intégrées
  • Utilisation propre de GitHub et du versionnage
  • Projet validé sans erreurs W3C HTML/CSS

🔧 Problèmes rencontrés et solutions

🔴 Problèmes rencontrés 🟢 Solutions trouvées
Pages mal alignées sur petit écran Utilisation de media queries pour ajuster la grille
Animations trop rapides ou saccadées Réglage des timings avec ease-in-out et durations adaptées

Axes d'amélioration identifiés

🎯 Problèmes non résolus et améliorations à apporter

Formulaires non fonctionnels (pas de base de données) Créer une version avec backend (PHP ou Firebase)
Contenu textuel parfois trop court Enrichir avec des articles ou vidéos complémentaires

Conseils pour refaire ce projet

  • Utiliser Flex et Grid dès le début pour éviter les conflits de layout
  • Valider régulièrement le HTML/CSS au fil du projet
  • Organiser les commits par fonctionnalité, avec des messages clairs
